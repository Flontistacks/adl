# Testing Standards

## Overview

Test real behavior where it adds value. Skip trivial assertions.

## Testing Framework

**Framework**: Go standard `testing` package
**Runner**: `go test ./...`

## Test Types

| Type | Tool | Location | When to Use |
|------|------|----------|-------------|
| Unit | `testing` | `internal/*/*_test.go` | Download type detection, config parsing, RPC message parsing |
| Integration | `testing` + test aria2c | `internal/aria2/*_test.go` | RPC client (if aria2c available in CI) |

## Coverage Requirements

**Target**: No hard minimum for v1
**Enforcement**: Manual review

**Critical paths that MUST have coverage:**
- URL/magnet/torrent path auto-detection
- Config load/save defaults

## Running Tests

```bash
# Run all tests
go test ./...

# Run with coverage
go test -cover ./...

# Run specific package
go test ./internal/download/...
```

## Mock Strategy

**Approach**: Interface-based mocking for aria2 RPC client in unit tests

**Guidelines**:
- Don't mock Bubble Tea views unless necessary
- Test download detection logic with table-driven tests
